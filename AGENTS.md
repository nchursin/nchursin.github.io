# AGENTS.md

## О проекте

Это персональный сайт-визитка Никиты Чурсина: `https://chursin.dev`.

На сайте размещены:

- краткая информация об авторе;
- навыки, опыт и образование;
- собственные и open-source проекты;
- публикации и блог;
- корпоративные тренинги.

Основной язык пользовательского контента — русский.

## Стек и публикация

Сайт собран на Hugo и использует тему `resume` из git-подмодуля
`themes/resume` (`eddiewebb/hugo-resume`). Конфигурация находится в
`config.toml`.

GitHub Actions собирает и публикует сайт в GitHub Pages при push в ветку
`master`. Workflow использует Hugo Extended 0.111.3 и рекурсивно получает
git-подмодули.

Перед локальной работой убедитесь, что тема инициализирована:

```sh
git submodule update --init --recursive
```

Основные команды:

```sh
hugo server
hugo
```

Также доступны цели `make build`, `make watch` и `make deploy`. Не запускайте
`make deploy` без прямого запроса пользователя: эта цель собирает архив и
вызывает внешний deployment-скрипт.

## Структура

- `content/` — страницы и публикации в Markdown.
  - `_index.md` — текст первого экрана главной страницы.
  - `blog/` — статьи; каждая крупная статья хранится в page bundle вместе с
    изображениями.
  - `projects/creations/` — собственные проекты.
  - `projects/contributions/` — вклад в open source.
  - `publications/` — доклады и публикации.
  - `trainings/` — описание корпоративных тренингов.
- `data/` — структурированные данные о навыках, опыте, образовании и
  сертификатах.
- `layouts/` — локальные шаблоны, перекрывающие одноимённые шаблоны темы.
  - `index.html` определяет порядок секций главной страницы.
  - `partials/nav.html` определяет пункты бокового меню.
  - `partials/sectionSummary.html` — общий анонс контентных секций.
  - `partials/trainingsSummary.html` — компактный анонс тренингов на главной.
  - `trainings/section.html` — страница списка тренингов.
- `static/` — изображения и CSS, копируемые в собранный сайт без обработки.
- `themes/resume/` — внешний git-подмодуль. Не редактируйте его для изменений
  сайта; делайте override в корневом `layouts/` или `static/`.
- `public/` — результат локальной сборки. Каталог игнорируется Git и не должен
  редактироваться вручную.
- `resources/_gen/` — сгенерированные Hugo ресурсы. Не редактируйте вручную.

## Как устроена главная страница

Главная собирается в `layouts/index.html`. Видимость большинства секций
управляется флагами `show...` в `[params]` файла `config.toml`. Порядок пунктов
в `layouts/partials/nav.html` должен соответствовать порядку секций на главной.

Пункты меню на главной используют якоря (`/#skills`, `/#trainings` и т. п.).
Контентные секции дополнительно ведут на отдельные страницы, например:

- Open Source: `/#open` → `/projects/contributions/`;
- Тренинги: `/#trainings` → `/trainings/`.

При добавлении новой секции обычно нужны четыре изменения:

1. контент в `content/<section>/`;
2. feature flag в `config.toml`, если секцию можно отключать;
3. пункт в `layouts/partials/nav.html`;
4. summary-блок в `layouts/index.html`.

Если полная страница содержит много текста, не выводите `.Content` целиком на
главной. Создайте отдельный компактный partial, как для тренингов.

## Правила контента

- Сохраняйте русский язык и существующий деловой, прямой тон.
- Не придумывайте опыт, клиентов, показатели, отзывы, цены или доступные
  форматы. Уточняйте такие данные у владельца сайта.
- Для внутренних ссылок используйте корневые URL сайта (`/trainings/`) или
  Hugo permalink-поля в шаблонах.
- Изображения статьи размещайте рядом с `index.md`, если они принадлежат только
  этой статье. Общие изображения размещайте в `static/img/`.
- Не меняйте старые материалы и данные, если задача их не затрагивает.

## Проверка изменений

После изменения контента, конфигурации или шаблонов выполните:

```sh
hugo --destination /tmp/nchursin-site-check --cleanDestinationDir
git diff --check
```

Проверьте сгенерированный HTML нужной страницы и главной, особенно:

- порядок и адреса пунктов меню;
- наличие и уникальность `id` секций;
- внутренние ссылки;
- отображение Markdown и изображений;
- отсутствие полной версии контента внутри краткого анонса.

В текущей версии проекта Hugo может показывать предупреждения темы о deprecated
API, а также предупреждение о raw HTML в старом `content/projects/_index.md`.
Не считайте их следствием новой правки, если их количество и содержание не
изменились.

Перед передачей результата просмотрите `git status --short` и не включайте в
работу несвязанные пользовательские изменения.
